# @alova/wormhole 官网内容规划方案

**核心 Slogan：消除你和服务端之间的一切距离**

---

## 一、核心定位

两条并行主线：

**「消除中间文档」** 吸引传统开发者——后端交付接口后，你不需要再打开 Swagger，不需要在文档站和编辑器之间反复切换，接口信息直接出现在你的编辑器里。

**「AI Skill」** 吸引 Vibe Coding 用户——你的 AI 也看不到接口，wormhole 为它生成专属的 Skill 文档，让 AI 直接读懂你的接口，精准生成调用代码。

工具名"虫洞"天然契合"拉近距离"的定位，两条主线共用一个 Slogan，贯穿官网始终。

### 核心能力区

标题：**一次生成，人和 AI 都能用**

分两组呈现四张卡片：

**▍ 为你生成**

🔧 **调用函数** — 内置支持 alova / axios / fetch / ky，按需引入，Tree-shaking 友好。也支持完全自定义模板。

📐 **TypeScript 类型** — 请求参数与响应结构全自动推导，零手写，JavaScript 项目同样兼容。

📖 **编辑器内嵌文档** — 配合 VSCode 扩展，直接在编辑器检索 API、查看完整参数表。后端接口变更时**自动重新生成**，始终保持最新状态，彻底告别切换 Swagger。

**▍ 为你的 AI 生成**

🤖 **AI Skill 文档** — 为每个接口生成 AI 可精准理解的 Skill 描述，包含参数结构、响应类型、调用示例和引入路径。Cursor / Copilot / Claude 读取后直接生成可运行代码，不再猜测，不再出错。★ Vibe Coding 时代的标配。

---

### AI Skill 专题区

标题：**Vibe Coding 时代，AI 也需要读懂你的接口**

左侧文案：

> 你让 AI 帮你写接口调用代码，它却总是猜错参数名、搞错响应结构、引入路径写错——不是 AI 不够聪明，而是它根本看不到你的接口文档。
>
> wormhole 为每个接口生成一份 AI 专属的 Skill 文档，专门为 AI 设计的结构化描述：参数从哪里传、响应怎么用、代码该怎么写、从哪里引入。AI 读完，直接给你可运行的代码。

右侧对比展示：

```
❌ 没有 Skill 文档

你：帮我调用获取用户信息的接口
AI：
  import axios from 'axios'
  axios.get('/user?id=1')
  // 参数结构猜错
  // 引入路径不对
  // 响应类型缺失

─────────────────────────────

✅ 有 wormhole Skill 文档

AI 读取 Skill →
  知道函数名、参数结构、引入路径
AI：
  import { getUserInfo } from './api/user'
  getUserInfo({
    pathParams: { accountCode: 'U001' }
  })
  // 一次正确，可直接运行
```

## 三、介绍页规划

### 产品定位段落

> @alova/wormhole 是一个 OpenAPI 代码生成工具。它的目标不只是生成代码——而是彻底消除前后端协作中的摩擦点，让接口信息以最自然的方式出现在开发者面前。你不需要打开 Swagger，不需要手动复制参数，你的 AI 也不需要靠猜。

---

### 它改变了什么？（新旧流程对比）

|            | 旧流程                                                               | 新流程（人工开发）                                    | 新流程（AI 辅助）                                      |
| ---------- | -------------------------------------------------------------------- | ----------------------------------------------------- | ------------------------------------------------------ |
| 接口交付后 | 打开 Swagger → 复制参数 → 切回编辑器 → 手写代码 → 再告诉 AI 接口细节 | `npx alova gen` → 编辑器直接查文档 → 直接调用生成函数 | `npx alova gen` → AI 读取 Skill → 直接生成正确调用代码 |
| 接口变更后 | 重复以上全部流程                                                     | VSCode 扩展自动重新生成，无需任何操作                 | AI Skill 自动更新，AI 始终知道最新接口                 |

---

### 为 AI 时代设计的接口文档

> 当你使用 Cursor 或 Copilot 写代码时，AI 对你项目的接口一无所知。它不知道参数叫什么、响应长什么样、该从哪里引入。你需要手动粘贴接口信息给它，本质上还是在"查文档"。
>
> wormhole 的 AI Skill 文档从根本上解决这个问题。生成后，AI 工具可以直接检索并读取每个接口的完整描述，精准完成调用代码的生成。**人和 AI，共享同一份接口上下文。**

---

### 四种产出物详解

**🔧 调用函数**（为你生成）
内置 alova / axios / fetch / ky 四套模板，按 tag 文件拆分，支持 Tree-shaking。支持完全自定义 Handlebars 模板，生成文件结构与模板结构完全一致，灵活可控。

**📐 TypeScript 类型**（为你生成）
自动推导请求参数与响应数据类型，开启 `strictNullChecks` 即可获得完整类型提示。JavaScript 项目同样兼容。

**📖 编辑器内嵌文档**（为你生成）
配合 VSCode 扩展，在编辑器内直接检索 API、查看完整参数表与响应结构。后端接口有变更时**自动重新生成**，始终保持最新状态，彻底告别 Swagger 切换。

**🤖 AI Skill 文档**（为你的 AI 生成）
为每个接口生成结构化的 AI 可读描述，包含参数定义、响应类型、调用示例和 import 路径。Cursor / Copilot / Claude 读取后直接生成可运行代码。支持自定义 Skill 模板，适配不同 AI 工具的格式需求。接口变更时同步自动更新。

---

### 始终保持最新状态

配合 VSCode 扩展，wormhole 自动检测 OpenAPI 文档变更并触发重新生成。无需手动执行任何命令——调用函数、类型、编辑器文档、AI Skill 全部自动同步，让你的前端代码始终与服务端接口保持一致。

---

### 灵活性与扩展性

**自定义模板** — 基于 Handlebars 模板引擎，自由定义生成文件的结构和内容。支持按 tag 拆分文件、partial 引入、不覆盖文件标记等机制。

**插件系统** — 完整的生命周期 Hook，可在任意阶段介入修改生成结果，包括直接修改生成文件内容。

**修改生成内容** — 通过内置 rename 插件修改函数名，通过 importType 插件注入外部类型，完全掌控输出。

**生成进度可观测** — 内置 onProgress 回调，CLI 多行进度条展示，便于 CI/CD 集成。
