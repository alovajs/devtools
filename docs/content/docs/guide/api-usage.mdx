## 选择模板风格

根据使用的模板不同，访问生成 API 的方式也不同。

### functional 模板（按需引入，推荐）

每个 API 作为独立函数从对应 tag 文件导出，支持 Tree-shaking。

```typescript
import { getUser } from './api/user';
import { createArticle } from './api/content';

const user = await getUser({ params: { id: 1 } });
const article = await createArticle({ data: { title: 'Hello' } });
```

### alovaGlobals 模板（全局挂载）

通过 `Apis` 变量访问（可通过 `global` 配置自定义变量名）。

在 `main.[js/ts]` 入口文件中引入自动生成目录下的 `index.[js/ts]`：

```js title="main.js"
import './your-generating-api-folder';
```

或在需要使用的文件中导入 `Apis` 变量：

```js
import { Apis } from './your-generating-api-folder';
```

调用 API 时，可以通过 `params`、`pathParams`、`data`、`headers` 来指定请求参数：

```js
useRequest(() => Apis.user.changeProfile({
  // （可选）query 参数
  params: { id: 12 },
  // （可选）path 参数
  pathParams: { id2: 20 },
  // （可选）body 参数
  data: { name: 'alova', age: 18 },
  // （可选）header 参数
  headers: { 'Content-Type': 'application/json' },
  // 其他 method 支持的 config 配置项
  cacheFor: 100 * 1000,
  transform: response => response.detail,
}));
```

## 设置 alova 参数

使用 alova 模板时，可以在 `${output}/index.[js/ts]` 中设置 `createAlova` 的全局参数。当重新生成代码时，此文件**不会被覆盖**。

`index` 文件内容取决于使用的模板：

- **functional 模板**：入口文件直接创建 `alovaInstance`，所有 tag 文件的 API 函数都引用此实例
- **alovaGlobals 模板**：入口文件创建 `alovaInstance`，然后构建全局 `Apis` 对象并挂载

你可以在 `createAlova` 中像往常一样编写拦截器、更换请求适配器。

## 旧项目迁移

如果希望在已使用 alova 的项目中集成 VSCode 扩展，按以下步骤进行：

1. **根据 OpenAPI 规范文件生成代码** — 运行 `npx worma gen`。
2. **替换 alova 实例** — 将 `${output}/index.[js/ts]` 中的 alova 实例用原项目中的 alova 实例代码替换。
3. **修改引入路径** — 在项目中已定义好的 API 调用函数中，修改 alova 实例的引入路径为 `${output}/index.[js/ts]`。

这样可以在不改变原始代码的情况下集成自动生成的代码。

## 注意事项

1. **`strictNullChecks`** — 在 TS 项目中，如果发现 VSCode 无法正确智能提示，请在 `tsconfig.json` 中设置 `"strictNullChecks": true`。
2. **API 提示为 `any` 类型** — 可按以下步骤排查：
   - 确认此 API 是否在入口文件中引入。
   - 重启 VSCode。
