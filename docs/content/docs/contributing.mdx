感谢你对 worma 的关注！worma 是一个完全开源、由社区驱动的项目，我们欢迎任何形式的贡献——无论是代码、文档、Issue 反馈还是经验分享。

## 开始之前

- 在 [GitHub Issues](https://github.com/alovajs/devtools/issues) 中搜索是否已有相关的讨论或需求。
- 对于较大的功能改动，建议先开一个 Issue 与维护者沟通方案，避免重复劳动。
- 查找带有 `good first issue` 标签的任务，适合初次贡献者上手。

## 开发环境

worma 使用 pnpm 管理 monorepo，推荐使用 Node.js ≥ 18 与 pnpm ≥ 8：

```bash
# 安装依赖
pnpm install

# 启动文档站点（开发模式）
pnpm --filter docs dev

# 运行单元测试
pnpm test
```

## 提交规范

我们使用 [Conventional Commits](https://www.conventionalcommits.org/) 规范，并由 commitlint 校验提交信息。常用类型：

- `feat`：新增功能
- `fix`：修复缺陷
- `docs`：文档变更
- `refactor`：重构（无功能变化）
- `test`：测试相关
- `chore`：构建 / 工具链调整

示例：

```bash
git commit -m "feat(core): 支持 oneOf 联合类型生成"
```

## Pull Request 流程

1. Fork 仓库并基于 `main` 分支创建特性分支：`feat/your-feature`。
2. 确保通过 `pnpm test` 与 lint 检查。
3. 提交 PR 时清晰描述改动动机、范围与验证方式。
4. 等待维护者 Review，根据反馈调整。

## 文档贡献

文档位于 `docs/content/docs`，使用 MDX 编写。你可以：

- 修正错别字、补充示例代码
- 翻译或完善多语言内容
- 补充使用场景与最佳实践

## 行为准则

请保持友善、包容的沟通氛围，尊重每一位贡献者。我们致力于打造对所有人友好的开源社区。
